### components versions

* supervisor. 4.0.0     

* web. 3.0.0

* sync. 0.1.0      

### Web UI

This release included a los Features and Fixes. We will enumerate only the main changes.

#### GENERAL FEATURES

* The version of every componente was added into the Help tab

* Tutorials fixed and improved

* Customer name is no longer required to create customers. is autogenerated

* Many dialogs and anoying confirmation dialogs removed. Using an alert insted.

* Informative alert in the bottom right of the screen.

* Agent for Windows. Installation script in Windows Task Managment

* on-premises. user administration features added

* Task and workflow import unified

* Dashboard empty tabs introductory screen

#### TEMPLATES FEATURES

* Template name is autogenerated using the Host name.

* FIX: Files assignment and reusing

#### WORKFLOWS FEATURES

* Export to File

* Import from File

* Copy to New

* Improved Task Creation.

* When assigning tasks to workflows a new copy of the selected task is created

* Guided workflows import. Auto assign Bot Host and Runas

#### TASKS FEATURES

Many validations and Visual improvements were added.

* FEATURE: Web request task will populate next task inputs with the response. Response body, headers and status are captured

* FIX: Fixed arguments and Environment variables are being checked before creation.

* FIX: When creating tasks the file can be included with the Task Payload.

#### FILES FEATURES

* FIX: Files are shared between Tasks, Monitores and Templates.

* FIX: Smart files location added. If a file exists will be used to avoid duplicates.


### Supervisor Api

* FEATURE: The Engine now implements a global state handler. Files will trigger events, notification and logger for every crud operation.

* FEATURE: Workflow serialization. Import and export serialized information

* FEATURE: Templates serialization. Import and export serialized information

* FEATURE: Tasks serialization. Import and export serialized information

* FEATURE: Files serialization. Import and export serialized information

* FEATURE: Files locate. Identifying existent files by checksum, size, mimetype allowes to reuse file in many tasks

* FEATURE: Workflow Events are created dynamically.

* FEATURE: Every task will have its own fingerprint. Also versioning is posible

### Gateway Api

* FEATURE: Customer name is now a Key and is autogenerated. Using display_name instead

## Apr 20, 2021

### components versiones

* supervisor. 2.4.0   

* web. 2.4.0    

* gateway. 1.4.0    

* auth. 0.0.1     

* sync. 0.0.3      

### Main features.

#### on-premises

* LDAP/AD Login. Users profile creation and verification can now be performed in case-insensitive manner.

Default group (configuration req.)
User match/create by email, username (case insensitive)

* Toggle email notifications for activation, customerInvitation, invitation, passwordRecover, registration.

* Toggle registration form, google login options, password reset form.

* Can customize login form inputs placeholders 


#### cross env (saas, on-premises)

* Cloud Domain Enterprise login (Azure AD integration)

* Notifications can now be sent using email and username.

* Dynamic approvers for Approval tasks. Approvers are set during task Triggering and should be defined in the previous task of a workflow or from the triggering task.

* ACL's and dynamic ACL's for Task and Workflows. This allows assignment of the workflow/task (job) execution to particular users. The ACL's must be set during task triggering.

* Task Arguments can now be set as Optional

* Core Arguments as JSON encoded (optional). The core will JSON.encode all task arguments and JSON.decode for each argument will be required within the scripts. Future releases will perform this process automatically.

* Added Task input type JSON. Validation only applicable for user input.

* Approval Task parametrization: success_enabled, failure_enabled, cancel_enabled, ignore_enabled 

* Task properties can now be set dynamically. (working on the documentation)

* Username was added to env THEEYE_JOB_USER

* Can now dynamically set user_inputs, user_inputs_member

* Optional during triggers can be set via API and task output. (working on the documentation)

* Workflow Job table view. The jobs list is displayed aligned in columns, each filled in with the arguments received in the first step (task) of the workflow.

* New endpoints added in the Core API.

| method | path | description |
| ----- | ----- | ----- |
| GET | /workflow/:workflow_id/job |  Fetch all workflow jobs (fetch api ready) |
| GET | /workflow/:workflow_id/job/:job | Fetch a workflow job | 
| GET | /workflow/:workflow_id/job/:job/jobs | Fetch all task jobs withing a workflow job execution |
| PUT | /job/:job/assignee | Set user_inputs, user_inputs_members. also set job acl. body filter username, email. determine organization using session |
| PUT | /job/:job/acl | Set job acl |
| PUT | /workflows/:workflow/job/:job/acl | Set job acl by workflow |
| PUT | /workflows/:workflow/job/:job/cancel | Cancel workflow job execution |
